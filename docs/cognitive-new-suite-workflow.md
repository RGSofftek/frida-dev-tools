# Cognitive Workflow: Create New Suite

## Scope

This document maps the "New Suite" creation flow from inside an existing app (`/suites`) and the network requests required to reproduce it.

Captured on: 2026-04-30  
Environment: `https://cognitivetesting.online/suites`  
Selected app during capture: `Tmp-IterProbe-1777573320302`  
Created suite artifact: `Tmp-Suite-Mapping-20260430-1234` (`processId`: `5640573`)

## UI Workflow

1. Open app suites page (`/suites`).
2. Click `New Suite`.
3. In dialog `Enter your Suite name`, input suite name.
4. Click `OK`.
5. Wait for confirmation dialog (`Done` / `Suite created`).
6. Click `Ok`.
7. New suite appears in the list with a generated numeric process ID.

## Network Sequence Observed

Suite creation is implemented as Azure storage provisioning calls (not a single `suites-createSuite` API call).

### 1) Create suite root directory

- Method: `POST`
- URL: `https://us-central1-cognitive-testing.cloudfunctions.net/azure-createDirectory`
- Content-Type: `application/json`
- Body:

```json
{"path":"Processes/5640573"}
```

- Response: `true` (HTTP 200)

### 2) Create suite steps directory

- Method: `POST`
- URL: `https://us-central1-cognitive-testing.cloudfunctions.net/azure-createDirectory`
- Content-Type: `application/json`
- Body:

```json
{"path":"Processes/5640573/Steps"}
```

- Response: `true` (HTTP 200)

### 3) Create first step directory

- Method: `POST`
- URL: `https://us-central1-cognitive-testing.cloudfunctions.net/azure-createDirectory`
- Content-Type: `application/json`
- Body:

```json
{"path":"Processes/5640573/Steps/0"}
```

- Response: `true` (HTTP 200)

### 4) Seed default Actions file

- Method: `POST`
- URL: `https://us-central1-cognitive-testing.cloudfunctions.net/azure-createTxtFile`
- Content-Type: `application/json`
- Body:

```json
{"path":"Processes/5640573/Steps/0","file":{"name":"Actions.txt","content":"Hello World"}}
```

- Response: `true` (HTTP 200)

## Notes

- `processId` (`5640573` in this run) is generated by the frontend before directory creation and used as the folder key under `Processes/`.
- Requests observed here did not include an explicit `Authorization` header in DevTools output; origin was `https://cognitivetesting.online`.
- CORS preflight (`OPTIONS`) requests were issued for both Cloud Function endpoints.
- No separate suite-metadata endpoint appeared in XHR/fetch during this creation action; list updated after provisioning and confirmation dialog.
